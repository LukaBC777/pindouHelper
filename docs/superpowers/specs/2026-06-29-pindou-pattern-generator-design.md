# 拼豆图纸生成器 — 设计文档

- 日期：2026-06-29
- 状态：设计已确认，待评审
- 目标平台：Windows 桌面

## 1. 目标与背景

把一张图片转换成可照着拼的**拼豆图纸**：把图片像素化到拼豆板网格，每格匹配成最接近的豆子颜色，输出带坐标、色号和用量清单的图纸（参考 `template.png` 的样式）。

核心使用流程是**两阶段**：

1. **生成 + 编辑阶段**：导入图片 → 选板型 → 自动像素化成可编辑的色号网格 → 用户在预览上手动替换/调整个别格子的豆色。
2. **导出阶段**：用户调整满意后 → 渲染成最终可打印图纸（PNG / PDF）。

## 2. 技术选型

- **语言/GUI**：Python + PySide6（Qt）。
- **图像/数值**：Pillow + numpy。
- **色彩匹配**：CIE Lab 空间 + CIEDE2000 Delta-E 最近邻（不用 RGB 欧氏距离）。
- **打包**：PyInstaller 出 Windows 单文件 exe。

**为什么不是 CNN**：像素到豆色是确定性的最近邻颜色匹配问题，不需要训练数据，CNN 在此只会更慢、更差、更难维护。真正的工程量在网格化、色数压缩、图纸渲染与交互编辑。

## 3. 整体架构

算法层为纯函数，与 GUI 严格分离，可独立单元测试。GUI 是薄壳。

```
图片 → [sizing] → [quantize] → 色号网格 → [bom]    → 用量面板
                                  │        [render] → 预览 / 导出(PNG/PDF)
                                  ↑
                          [手动改色覆盖层]
```

## 4. 模块拆分

| 模块 | 职责 | 主要依赖 |
|---|---|---|
| `palette` | 加载某品牌调色板（色号 / 名称 / RGB / Lab / symbol），可切换品牌；提供查询接口 | 数据文件 |
| `sizing` | 输入图片宽高比 + 选定板型 → 计算豆子网格尺寸（含多板拼接），纯函数 | numpy |
| `quantize` | 核心：缩放 → 可选抖动 → Lab/Delta-E 最近邻匹配 → 可选最大色数压缩 → 输出 `int` 色号网格，纯函数 | numpy |
| `bom` | 统计每个色号的用量与总数，纯函数 | - |
| `render` | 把色号网格画成图纸：色块、每 10 格粗线、板边界线、四周坐标、顶部标题、底部色卡条；预览与导出共用 | Pillow |
| `editor_state` | 维护「计算网格 + 用户手动覆盖」的合并视图与撤销/重做 | - |
| `gui` | 打开图、控件面板、可缩放画布、点格改色、BOM 面板、导出按钮 | PySide6 |

每个模块边界清晰：`sizing`/`quantize`/`bom` 输入输出都是数组与数值，不依赖 GUI，可单独测试；`render` 接收网格 + 调色板产出图像；`gui` 只编排这些模块并处理交互。

## 5. 关键算法细节

### 5.1 颜色匹配
- 调色板每色预存 Lab 值（数据源已含，见第 8 节）。
- 每个目标格取色后转 Lab，对调色板做 CIEDE2000 Delta-E 最近邻，取色号。
- 为性能，调色板 Lab 预载为 numpy 数组，整图向量化匹配。

### 5.2 板型与自动尺寸
- 标准板尺寸：`15×15`、`29×29`、`52×52`、`52×104`（宽×高，单位：豆）。
- 用户选板型后，程序按图片宽高比计算最合适的拼板数（行数 × 列数），总网格 = 板数 × 板尺寸；用户可手动微调板数。
- 图纸渲染时画出**板边界线**（区别于每 10 格的计数粗线），方便分板拼装。

### 5.3 最大色数限制
- 先用完整调色板全量匹配，统计各色用量。
- 保留用量最高的 N 个色号，其余格子重映射到这 N 个色号中 Lab 距离最近的一个。
- 保证最终只用调色板中真实存在的颜色（不会凭空造色）。

### 5.4 抖动（dithering）
- 可选开关，**默认关闭**（动漫/卡通图关掉更干净）。
- 开启时对照片类图用 Floyd–Steinberg 抖动后再匹配。

### 5.5 手动改色
- 预览画布上点击/拖动选中格子，从调色板选一个新色号写入覆盖层。
- 覆盖层与计算网格合并后才参与 BOM 与导出。
- 支持撤销/重做。

## 6. 输出图纸（对齐 `template.png`）

- 色块网格，每格一个豆色。
- 每 10 格一条计数粗线 + 板边界线。
- 四周坐标数字（行/列）。
- 顶部标题（品牌 + 色数，如 "Mard221"）。
- 底部色卡条：每个用到的色号 + 用量。
- 放大时可选在格内叠加色号符号（symbol）。
- 导出 PNG 与 PDF。

## 7. 错误处理与测试

**错误处理**
- 非法/损坏图片：提示并中止。
- 超大网格：限制最大豆数上限，超出给出提示或自动降采样，防止卡死。
- 调色板文件缺失/格式错误：启动时校验并报错。

**测试**
- `sizing` / `quantize` / `bom`：纯函数，用小合成图 + 微型调色板做单元测试（含边界：极端宽高比、单色图、最大色数边界）。
- `render`：快照（snapshot）对比关键元素是否绘制。
- GUI：手动验证交互。

## 8. 前置数据任务（已查证，阻塞项已解除）

- **数据源**：开源仓库 [maxcleme/beadcolors](https://github.com/maxcleme/beadcolors) 提供 MARD 色表 CSV。
  - 原始文件：`https://beadcolors.eremes.xyz/raw/mard.csv`
  - v2 版本含预计算 Lab（`lab_l/lab_a/lab_b`），v3 含 `symbol`。
  - 字段：`reference_code`、`name`、`rgb_r/g/b`、`rgb_hex`、`lab_*`、`symbol`。
- **注意**：MARD 全集为 291 色，"Mard221" 为其中 221 色子集；星芒拼豆 221 色很可能对应该子集。
- **方案**：先以 MARD 数据为基准调色板跑通；星芒与 MARD 的精确色号映射作为后续校准任务。调色板设计为可插拔数据文件（每品牌一份 JSON/CSV），切换品牌即换文件。

## 9. 范围之外（YAGNI）

- 不做账号/云端/分享。
- 不做 CNN / 机器学习上色。
- 首版不做库存（「只用我有的颜色」）过滤——可由最大色数 + 手动改色近似覆盖，后续再说。
